Основні команди Git

git config -- global user.name ... - глобальний конфіг для локальних репозиторіїв, а саме ім'я користувача.

git config -- global user.email ... - глобальний конфіг для локальних репозиторіїв, а саме пошта користувача.

git init - створення локального репозиторія в активній папці.

git status - переглянути зміни і нововведення, які потрібно закомітити.

git commit - закомітити зміни в файлах і зберегти дану версію в історії.

git commit -m "" - створення локального репозиторія в активній папці з утворенням короткого месенджу без відкривання GUI.

git log - переглянути історію комітів в поточній гілці.

git checkout id - перехід до версії id, який отримаємо з виконання команди git log.

git checkout - повернення до найновішої версії.

git branch - перегляд гілок, з виділенням активної.

git push - завантаження змін з локального репозиторія у віддалений.

git pull - завантаження змін з віддаленого репозиторія у локальний.

git merge - злиття гілок в поточну.

git clone - завантаження віддаленого репозиторія з історією змін.

git pull - завантаження змін з віддаленого репозиторія у локальний.

git revert - скасування небажаного коміту.

git reset - відмінити коміт файлу зі збереженням вмісту.

git rebase - перезаписсати історію коммітів при злитті гілок.

git cherry-pick - перенесення необхідних комітів між гілками.

git stash - зберігає зміни активної директорії в комірку незавершенних змін +які можна повернути в любий момент часу.

.gitignore - файл, за допомогою якого git буде ігнорувати вказані файли або папки.
